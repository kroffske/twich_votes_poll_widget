# Дизайн анимации “весов”

## Визуальная идея

Зритель должен за 1–2 секунды понять:

- какой вариант лидирует;
- насколько большой разрыв;
- сколько голосов или баллов набрано;
- идет ли Poll прямо сейчас.

Формат “весов” работает лучше всего для баттлов 1v1: “левая сторона против правой стороны”.

## Поведение

### Балка

Балка наклоняется в сторону более “тяжелой” чаши. Наклон ограничен `maxTilt`, чтобы overlay не выглядел сломанным при большом разгроме.

```text
tilt = ((rightValue - leftValue) / (rightValue + leftValue)) * maxTilt
```

Пример:

- left=50, right=50 → 0°
- left=75, right=25 → примерно -8° при maxTilt=16
- left=100, right=0 → -16°

### Чаши

У каждой чаши есть вертикальное смещение:

```text
drop = ((choiceValue - opponentValue) / pairTotal) * 34px
```

Побеждающая чаша опускается, проигрывающая поднимается.

### Цифры

Числа анимируются через `requestAnimationFrame`, а не через CSS counters. Это дает контроль над округлением и форматированием больших чисел.

### Pulse при новом голосе

Когда значение choice выросло, overlay добавляет класс `pulse` к чаше. Анимация короткая и не мешает чтению текста.

### Weight chips

Маленькие “грузы” в чаше показывают относительный вес. Это декоративный элемент, но он помогает считывать разницу даже без чтения цифр.

## 3–5 вариантов

Для трех и более вариантов используется гибрид:

1. Сортируем варианты по выбранной метрике.
2. Топ‑2 идут на весы.
3. Все варианты показываются ниже в ranked list.

Так зритель видит главный конфликт и не теряет информацию по остальным вариантам.

## OBS рекомендации

- Размер Browser Source: 1920×1080.
- Для нижней плашки можно в OBS обрезать источник или масштабировать.
- FPS: 30 достаточно.
- CSS animation использует transform, поэтому нагрузка умеренная.
- При слабом ПК ставь `animation=reduced`.

Примеры URL:

```text
/overlay?token=...&mode=scale&metric=votes
/overlay?token=...&mode=scale&metric=points&maxTilt=12
/overlay?token=...&mode=bars&metric=votes&animation=reduced
```
